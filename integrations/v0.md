# Using Ramoira with v0

v0 generates UI components and copy. Attach your brand schema as context so generated copy matches your brand voice without manual briefing.

---

## Setup

**1. Generate a schema**

```sh
npx ramoira init
```

**2. Attach schema context**

For UI copy tasks, the published summary schema is typically sufficient:

```
https://ramoira.com/brands/[slug]/schema.summary.json
```

For tasks involving landing pages, pricing display, or claims, attach the full `brand.schema.json` to include commercial rules.

---

## Prompt pattern

When attaching the schema:

```
Use the attached Ramoira brand schema for all copy in this component.

Apply:
- identity.summary.neverDo — absolute constraints on every surface
- voice.approvedTones — match these; avoid voice.forbiddenTones
- voice.examples — the rejected examples define what this brand will never say
- commercial.pricing — check priceDisplayPermitted and discountPermitted before including any pricing copy
```

---

## Surface mapping

Match the v0 task to the closest `OutputSurface` to determine which constraints apply:

| v0 task | OutputSurface |
|:---|:---|
| Landing page hero | `paid_landing_page` |
| Product card copy | `product_detail_page` |
| Navigation / labels | `brand_narrative` |
| Social preview | `social_organic` |
| Email template | `email_acquisition` or `email_retention` |

The commercial and governance `surfaceRules` in the full schema contain surface-specific constraints for each of these contexts.
